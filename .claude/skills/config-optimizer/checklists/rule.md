# Rules チェックリスト

## スコープ

Claude Code Rulesのfrontmatter、globs設定、行数制限、CLAUDE.mdとの適切な役割分担を検証。

---

## Critical

### frontmatter必須項目
- [ ] frontmatter（`---`で囲まれた部分）が存在するか
- [ ] `globs` が設定されているか

---

## High

### globs設定
- [ ] `globs` パターンが具体的か（`**/*` のような広すぎるパターンを避ける）
- [ ] 対象ファイルが適切にマッチするか
- [ ] 否定パターン（`!`）の使用が適切か
- [ ] 複数パターンの場合、配列形式で記述されているか

### 行数制限
- [ ] ルールファイルが200行以下か
- [ ] 200行超の場合、分割が必要か検討

---

## Medium

### CLAUDE.md との役割分担
- [ ] 特定パスに限定できる内容がRulesにあるか（適切）
- [ ] プロジェクト全体に適用すべき内容がRulesにないか（→ CLAUDE.mdへ）
- [ ] CLAUDE.mdと重複する内容がないか

### 内容の適切さ
- [ ] ルールがパス固有の制約・規約を記述しているか
- [ ] 汎用的なプログラミング原則のみで構成されていないか
- [ ] 命令形で記述されているか

### 構造
- [ ] 明確なセクション分けがあるか
- [ ] 優先度の高い情報が上部にあるか
- [ ] 表や箇条書きを活用しているか

---

## Low

### 命名
- [ ] ファイル名が内容を適切に表しているか
- [ ] ハイフンケースで記述されているか

### スタイル
- [ ] 見出しレベルが一貫しているか
- [ ] 曖昧な表現を避けているか
- [ ] 具体例が含まれているか
