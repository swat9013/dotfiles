# Commands チェックリスト

## スコープ

Claude Code Commandsのallowed-tools最小権限、説明の明確さ、Skillsとの役割分担を検証。

---

## High

### allowed-tools制限
- [ ] 必要最小限のツールのみ許可しているか
- [ ] 全ツール許可（`*`）を避けているか
- [ ] 危険なツール（Write, Edit, Bash等）が本当に必要か確認
- [ ] 読み取り専用タスクに書き込みツールが含まれていないか

### 説明
- [ ] 明確な`description`があるか
- [ ] 引数がある場合、`argument-hint`があるか
- [ ] コマンドの目的が一目で理解できるか

---

## Medium

### 動的コンテキスト
- [ ] `!` プレフィックスが適切に活用されているか
- [ ] シェル出力の注入が効果的か
- [ ] 注入されるコンテキストが多すぎないか

### 名前空間
- [ ] サブディレクトリで整理されているか（多数ある場合）
- [ ] 命名が一貫しているか
- [ ] 名前から機能が推測できるか

### Skills との役割分担
- [ ] アクション系タスクがCommandになっているか
- [ ] 対話・相談系タスクがSkillになっているか
- [ ] 重複する機能がないか

---

## Low

### 冗長性
- [ ] 類似コマンドの重複がないか
- [ ] シンプルなタスクを複雑にしていないか

### model指定
- [ ] コスト最適化のため適切なモデルが指定されているか（haiku等）
