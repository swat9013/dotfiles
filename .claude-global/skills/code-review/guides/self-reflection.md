# コードレビュー検証ガイド

レビュー結果を出力する前に、各指摘を以下の観点で再検証する。
高信号基準を満たさないものは除外する。

## 検証項目

### 1. 位置検証
指摘した `ファイル:行番号` に該当コードが存在するか確認。

- 存在しない → 除外

### 2. 構文検証
修正案を適用した場合、構文エラーにならないか確認。

- エラーになる → 修正案を修正、または除外

### 3. 高信号基準チェック

**フラグすべき（残す）**:
- コンパイル/パースエラー、型エラー、missing imports
- 明確なロジックエラー（確実に間違った結果を生む）
- セキュリティ脆弱性（SQLインジェクション、XSS、認証・認可の欠陥）
- CLAUDE.md違反（具体的なルールを引用可能）

**フラグしない（除外）**:
- コードスタイルの懸念
- 入力依存の潜在的問題
- 主観的な改善提案
- Linterがcatchする問題
- 既存の問題（新規導入ではない）
- ベテランエンジニアがフラグしない細部

### 4. 重複排除
同一問題への複数指摘がないか確認。

- 重複あり → 最も適切な1つに統合

## 検証の原則

- 「これは本当に問題か？」と自問する
- 推測や潜在的問題は除外
- 根拠をコードから引用できないものは除外
- 迷ったら除外（偽陽性を避ける）
