# Review Claude Config

`.claude-global/` 配下の設定が `docs/claude-code-context-optimization.md` のガイドラインに従っているかレビュー。

## レビュー対象

$ARGUMENTS（省略時: 全体）

## レビュー観点

### 1. 構造整合性

**サイズ基準**:
| ファイル種別 | 上限 |
|-------------|------|
| CLAUDE.md | 150行 |
| rules/*.md | 200行 |
| SKILL.md | 500行 |

**確認項目**:
- [ ] 各ファイルがサイズ上限内か
- [ ] Rules に frontmatter（`paths:`）があるか
- [ ] 命名規則（kebab-case）が守られているか

### 2. 責務分離

**配置基準**:
| 内容 | 配置先 |
|------|--------|
| 全体原則（KISS等） | CLAUDE.md |
| パス固有ルール | rules/ |
| ワークフロー・手順 | skills/ |
| 並列実行・専門役割 | agents/ |
| 機械向け設定（hooks, permissions） | settings.json |

**確認項目**:
- [ ] パス固有でない内容が Rules にないか
- [ ] ワークフロー・手順が Rules にないか
- [ ] 機械向け設定が CLAUDE.md にないか

### 3. Progressive Disclosure

**確認項目**:
- [ ] SKILL.md に詳細が埋め込まれていないか（概要のみ）
- [ ] 詳細は `references/` に分離されているか
- [ ] 参照先が明示されているか

### 4. DRY原則

**確認項目**:
- [ ] 複数ファイルで同じ内容がないか
- [ ] CLAUDE.md と rules/ に重複がないか
- [ ] 2つ以上で使う内容は `_shared/` にあるか

### 5. 情報階層

**階層順（上から常時読み込み → 下はオンデマンド）**:
```
CLAUDE.md  → 常時読み込み（コスト高）
rules/     → パスマッチ時
skills/    → 必要時のみ
agents/    → コンテキスト分離
```

**確認項目**:
- [ ] 「この情報は常に必要か？」で階層選択が適切か
- [ ] より下位層に移せる情報がないか

### 6. 参照整合性

**確認項目**:
- [ ] 参照先ファイルが存在するか
- [ ] パス記載が正確か
- [ ] 古い参照が残っていないか
