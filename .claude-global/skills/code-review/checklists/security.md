# セキュリティレビューチェックリスト

## スコープ

攻撃者の視点でセキュリティ脆弱性を検出する。OWASP Top 10を中心に、認証・認可・機密情報の取り扱いを厳密にチェック。

---

## Critical

### インジェクション
- [ ] SQLインジェクション対策（パラメータバインディング）
- [ ] XSS対策（サニタイズ、エスケープ）
- [ ] コマンドインジェクション対策（シェル実行の回避）
- [ ] パストラバーサル対策（ファイルパスの検証）

### 認証・認可
- [ ] 認証の適切な実装（多要素認証の検討）
- [ ] 認可の適切な実装（権限チェック漏れがないか）
- [ ] セッション管理（固定化攻撃対策、適切な有効期限）
- [ ] CSRF対策

### 機密情報
- [ ] ハードコードされた認証情報がないこと
- [ ] 機密情報のログ出力禁止
- [ ] 適切な暗号化（アルゴリズム、鍵管理）

## High

### サーバーサイド
- [ ] SSRF対策（外部URLの検証）
- [ ] 安全でないデシリアライゼーションの回避
- [ ] ファイルアップロードの検証（拡張子、MIME、サイズ）

### 依存関係・設定
- [ ] 依存関係の脆弱性チェック（npm audit, Dependabot等）
- [ ] セキュリティヘッダーの設定（CSP, X-Frame-Options等）
- [ ] エラーメッセージで内部情報を漏洩していないか
