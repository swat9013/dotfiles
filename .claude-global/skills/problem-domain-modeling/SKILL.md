# Problem Domain Modeling

**このモデリングセッションはユーザーが終了を宣言するまで、反復ループを継続する。**

## 起動手順

1. **モデルファイルを作成する**
   - パス: `.claude/domain-models/YYYY-MM-DD-HHMMSS-{topic}.md`
   - `{topic}` は `$ARGUMENTS` から英数字+ハイフン30文字以内で生成
   - 初期内容は `references/output-schema.md` のテンプレートに従う（`status: open`, `loop_count: 0`）

2. **セッション開始を宣言し、課題の初期理解を示す**
   - `$ARGUMENTS` から読み取れる課題の概要を自分の言葉で要約する
   - 認識が正しいか、開始前にユーザーに確認しない（まず語彙採取から始める）

3. **最初の語彙採取質問をユーザーに返す**
   - 例: 「この課題で日常的に使っている言葉を3〜5個挙げてください」
   - 専門用語・略語・チーム固有の言い回しも歓迎することを伝える

4. **ユーザーの応答を待ち、反復ループへ進む**

## 反復ループ

各ループは以下の4フェーズで構成される。ループNはセッション開始から数えた回数。フェーズとセクションの対応は各フェーズの **出力** 項目で個別に指定する。

### ループN: 語彙採取

- **目的**: 一次源から語彙を浅く広く収集する
- **方法**: ユーザーが自然に使う言葉をそのまま記録する。定義や解釈は加えない。「それはどういう意味ですか？」は最小限に留める
- **出力**: 語彙リスト（語 + ユーザーの文脈説明）をモデルファイルの `## ループN — 語彙採取` セクションに書き込む

### ループN: モデル化

- **目的**: 採取した語彙を関係ネットワークとして整理する
- **方法**: 以下の関係パターンで語彙間のエッジを抽出する
  - 「AはBを含む」「CはDを引き起こす」「EとFは競合する」「GはHに依存する」
- **出力**: 関係ネットワーク（エッジリスト形式）をモデルファイルの `## ループN — モデル（関係ネットワーク）` セクションに書き込む
- **ユーザーへの確認**: 「この関係図はあなたの認識と合っていますか？」

### ループN: 歪み検証

> `[NEEDS DOGFOODING]`: 自律的な敵対検証プロトコルは dogfooding 後に実装予定

**暫定実装（dogfooding 前）:**

- **目的**: モデルの歪み・欠落・矛盾を発見する
- **方法**: ユーザーへの問いかけで検証する
  - 「このモデルで違和感を感じる箇所はありますか？」
  - 「見落としている重要な語彙はありますか？」
  - 「矛盾していると感じる関係はありますか？」
- **出力**: 検出した歪みを `## ループN — 歪み記録` セクションに書き込む（形式: `- [MISSING|CONFLICT|AMBIGUOUS] 説明`）

### ループN: 訂正と引き継ぎ

- **目的**: 歪み検証の結果をモデルに反映し、次ループへの足がかりを作る
- **方法**:
  - `loop_count` をインクリメントする
  - 訂正後のモデルは次ループの語彙採取・モデル化フェーズで反映する。現ループのセクションは追記せず歴史として保持する
- **出力**:
  - 訂正後のモデル → 次ループの `## ループN+1 — モデル（関係ネットワーク）` に反映
  - 引き継ぎ事項 → 現ループの `## ループN — 次ループへの引き継ぎ` に記録
- **ユーザーへの確認**: 「次のループで重点的に掘り下げたい領域はどこですか？」または継続意思を確認する

## 停止条件

ユーザーが以下のいずれかを宣言したとき、ループを終了する:

- 「終了」「閉じる」「close」「モデルを確定する」「このループで十分」等

**終了時の処理:**

1. モデルファイルの `status: open` を `status: closed` に更新する
2. モデルファイルの末尾に `## セッションサマリ` セクションを追記し、以下を記録する:
   - 採取語彙数
   - モデル化した関係数
   - 未解決の歪み（0件の場合も `0件` と記載）
   - 完了ループ数
3. 成果物パスをユーザーに案内する

自律的停止（飽和・収束による）は実装しない。ユーザーの宣言のみが停止条件。
