# グローバル CLAUDE.md ガイド

## 制約

| 項目 | 値 |
|------|-----|
| 公式上限 | 500行 |
| 推奨サイズ | 150行以下（超えると遵守率低下） |
| 内容 | 全プロジェクト共通の方針・環境制約 |

## グローバル vs プロジェクト CLAUDE.md

| 判断基準 | グローバル | プロジェクト |
|---------|-----------|------------|
| 適用範囲 | 全プロジェクト | 単一プロジェクト |
| 内容 | 普遍的な原則・環境制約 | プロジェクト固有情報 |
| 例 | 「KISS優先」「macOS環境」 | 「React構成」「API設計」 |

## CLAUDE.md vs README.md

| 観点 | CLAUDE.md | README.md |
|------|-----------|-----------|
| 対象読者 | Claude（AI） | 人間（開発者） |
| 目的 | 行動指示・制約 | プロジェクト説明 |
| 重複回避 | `@README.md`で参照、転記しない | - |

## 含めるべき内容

- コーディング哲学（設計原則、実装指針）
- 環境制約（OS、ツール、パッケージマネージャー）
- インタラクションルール（質問方針、オプション提示方法）
- 主要コマンド（ビルド、テスト、実行可能形式で）

## 避けるべき内容

- プロジェクト固有の情報 → プロジェクトのCLAUDE.mdへ
- パス固有のルール → rules/へ
- 手順やワークフロー → skills/へ
- `/init`生成のデフォルト内容をそのまま使用

## 書き方

- 文章より箇条書き・テーブルを優先
- コマンドは実行可能な形式（`npm run dev`）で記載
- 「〜すること」「〜しないこと」のアクション指向

## @import構文

```markdown
@README.md              # 同階層ファイル参照
@docs/architecture.md   # パス指定
```

- 再帰インポート最大5ホップ
- 大きなファイルを丸ごとインポートするより、要約をCLAUDE.mdに書く方が効率的

## CLAUDE.local.md

- 個人用プロジェクト設定（自動で`.gitignore`に追加される）
- CLAUDE.mdの後に読み込まれ、設定を上書き可能
