# CLAUDE.md チェックリスト

## スコープ

CLAUDE.mdの内容の適切さ、行数、構造、一貫性を検証。

---

## High

### 内容の適切さ
- [ ] プロジェクト固有の情報を含んでいるか
- [ ] 汎用的なプログラミング原則のみで構成されていないか
- [ ] Claudeが既知の一般知識を重複説明していないか
- [ ] 具体的なコマンド、構造、制約が記載されているか

### 行数
- [ ] 300行以下か（推奨200行以下）
- [ ] 長い場合、`docs/`等への分離を検討すべきか
- [ ] Progressive Disclosureが適用されているか

---

## Medium

### 構造
- [ ] 明確なセクション分けがあるか
- [ ] 優先度の高い情報が上部にあるか
- [ ] 読みやすい形式（表、箇条書き）を活用しているか
- [ ] 見出しレベルが一貫しているか

### 一貫性
- [ ] グローバルとプロジェクトで矛盾がないか
- [ ] Skills/Commandsの指示と整合しているか
- [ ] settings.jsonの設定と矛盾がないか

### プロジェクト情報
- [ ] リポジトリ概要があるか
- [ ] アーキテクチャ説明があるか
- [ ] 重要なコマンド・スクリプトが記載されているか
- [ ] ディレクトリ構造の説明があるか

---

## Low

### スタイル
- [ ] 命令形で記述されているか
- [ ] 曖昧な表現を避けているか
- [ ] 具体例が含まれているか

### 重複
- [ ] README.mdと重複する内容が最小限か
- [ ] Claude専用の情報に絞られているか
