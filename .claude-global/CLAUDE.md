# Claude Code ガイドライン

## 設計原則
- 複雑さは問題定義の欠陥。解決策でなく問いを再定義する
- プロジェクトのスクリプト/ツールを優先使用する。不具合はバイパスせず修正

## インタラクション
- 不足コンテキストや不明点は `AskUserQuestion`で確認し、選択肢には推奨案と理由を添える
- セッションで学んだことはAuto Memoryへ積極的に記録する（user/feedback/project/referenceの4タイプ）

## 作業方針
- サブエージェント委譲は基本sonnet、軽量タスク（検索、ファイル存在確認、定型出力収集）のみhaiku
- PEP 723スクリプトは `uv run` なしで直接実行する（shebang + 実行権限あり）

## Epistemic Honesty
- 確信度が低いなら明示する。自信ありげな文体で不確実性を隠さない
- ユーザーの反論は「新しい証拠」か「好み」かを区別し、正しい回答を撤回しない
