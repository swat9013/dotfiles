# 4層保存先判断基準

知見の保存先を判断するための基準。14カテゴリ分類は不要—保存先を直接判断する。

## CLAUDE.md

- プロジェクト全体に影響する方針・制約・アーキテクチャ決定
- 頻繁に参照が必要な情報（常時ロードされるため）
- Gotchas（落とし穴）で再発防止が必要なもの
- 例: 技術スタック選択理由、ディレクトリ構造の意図、環境固有の制約

## rules/

- 特定のファイルパターンに紐づくガイドライン
- 該当パスのファイルにアクセスした時だけ必要な情報
- 例: `*.test.*` 向けのテスト規約、特定ディレクトリの命名規則
- **必須**: 対象パスのglobパターンを明示すること

## skills/

- 繰り返し実行されるワークフローの改善
- 既存スキルへの手順追加・修正
- 新規スキル候補（実装は /managing-skills に委譲）

## settings.local.json

- コマンドの allow/ask/deny 権限設定
- ツール実行時に毎回承認が必要で自動化すべきもの
- 例: `uv run pytest` を allow に追加

## 破棄基準

以下に該当する知見は保存しない:
- 汎用的なベストプラクティス（プロジェクト固有でない）
- 一時的な作業メモ・デバッグログ
- 既存コンテキストと重複する内容
- コードから直接読み取れる情報
