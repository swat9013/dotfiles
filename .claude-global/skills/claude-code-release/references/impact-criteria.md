# 影響度判定基準

## 高影響（必ず抽出）

### 1. 破壊的変更・挙動変更（最優先）

既存の動作が変わり、ユーザーの設定やワークフローに影響するもの。

判定ポイント:
- 既存設定値が無効化・検証エラー化する
- デフォルト動作が変わる
- APIやCLIインターフェースの変更

### 2. 主要新機能

UI・ワークフロー全体に関わる新しい機能。

判定ポイント:
- 新しいコマンド、スラッシュコマンド、権限モデルの追加
- ユーザーの操作体験が変わるもの
- 環境変数やフラグの追加だけのものは除外

### 3. データ損失・クラッシュ系バグ修正

ユーザーのデータやセッションが失われる可能性があったバグの修正。

判定ポイント:
- 無限ループ（APIコール消費、CPU占有）
- OOM / クラッシュ
- ファイル破損・データ消失
- セッション喪失

### 4. 拡張機能への影響

スキル・hooks・rules・settings.jsonの仕様に関わる変更。

判定ポイント:
- スキルfrontmatterのフィールド追加・廃止・挙動変更
- hooks APIの変更（新イベント、引数変更）
- settings.jsonスキーマの変更（新キー、バリデーション変更）
- rulesのglob/paths挙動変更
- allowed-tools の対象ツール追加・削除

### 5. 顕著なパフォーマンス改善

計算量のオーダーが変わるレベルの改善。

判定ポイント:
- O(n^2) → O(n) 等のアルゴリズム改善
- 長時間セッションでの劣化修正
- マイナーチューニングは除外

## 除外（抽出しない）

| カテゴリ | 理由 |
|---------|------|
| 内部実装の最適化 | ユーザーに直接影響しない |
| 特定OS/環境のみの修正 | 影響範囲が限定的 |
| UX微調整（バッジ、ホバーテキスト等） | 操作体験を本質的に変えない |
| 一時的機能（April Fools等） | 永続的な変更ではない |
| テスト・CI修正 | ユーザーに影響しない |

## 境界ケースの判断指針

- `Changed` でも表示のみの変更 → 除外
- `Fixed` でも無限ループやクラッシュ → 必ず抽出
- `Added` でも環境変数1つの追加 → 原則除外。ただし描画・入力・パフォーマンス等の開発体験（DX）を変えるオプト・イン機能は抽出
- セキュリティ修正 → 内容次第だが、ユーザーアクション必要なら抽出
