# テスト品質レビューチェックリスト

## スコープ

テストコードの品質を検証。Khorikov 4本柱に基づき、リファクタリング耐性と実践的なテスト設計を重視。

用語定義は `references/testing-glossary.md` を参照。

---

## High

### テスト設計（Khorikov 4本柱）
- [ ] 観察可能な振る舞いを検証しているか（実装詳細ではなく）
- [ ] リファクタリング耐性があるか（正当なリファクタリングで壊れないか）
- [ ] Mockは境界（管理されていない依存）にのみ使用しているか
- [ ] Stubの呼び出しを検証していないか（verify禁止）

### テストカバレッジ
- [ ] 正常系のテストがあるか
- [ ] エラーケース・異常系のテストがあるか
- [ ] 境界値のテストがあるか

## Medium

### テストの品質
- [ ] テストが独立している（他のテストに依存しない）
- [ ] テスト名が何をテストしているか明確（Given-When-Then等）
- [ ] セットアップ/ティアダウンが適切
- [ ] テストデータが明確で再現可能

### テスト構成
- [ ] ユニットテストと統合テストが適切に分離されているか
- [ ] フレイキーテスト（不安定なテスト）がないか
- [ ] テストがビジネス要件と紐づいているか
