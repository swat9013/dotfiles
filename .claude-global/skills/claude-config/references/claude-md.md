## 配置場所と読み込み優先順位

複数ロケーションをサポートし、優先度順に**全て読み込まれる（加算的）**。

| 優先度 | パス | 用途 | 共有 |
|--------|------|------|------|
| 1（最高）| `/Library/Application Support/ClaudeCode/CLAUDE.md`（macOS）| エンタープライズポリシー | 組織全体 |
| 2 | `~/.claude/CLAUDE.md` | 個人の全プロジェクト共通設定 | 自分のみ |
| 3 | `./CLAUDE.md` または `./.claude/CLAUDE.md` | プロジェクト設定（git 管理） | チーム |
| 4 | `./.claude/rules/*.md` | パス条件付きルール | チーム |
| 5（最低）| `./CLAUDE.local.md` | 個人プロジェクト設定（`.gitignore` 自動追加） | 自分のみ |

**補足**: 公式上限は 500 行だが、Claude が確実に遵守できる実用上限は 150〜200 行。理想は **50 行以下**。IFScale 研究では 150〜200 指示で primacy bias（先頭偏り）が顕著になり、150 から遵守率が低下。Claude Code システムプロンプト自体が約 50 指示を含むため、ユーザー CLAUDE.md 100 行で計 150 指示に達する。

**50行構造の目安**:

| 割当 | 行数 | 内容 |
|------|------|------|
| オリエンテーション | ~5行 | プロジェクト概要・技術スタック |
| ルーティング | ~5行 | テストパス、ADR場所、linterコマンド |
| 禁止事項 | ~15行 | 明示的な制約（各項目にADR/ルール参照付き） |
| コマンド | ~10行 | ビルド・テスト・デプロイの最小限コマンド |
| リンク | ~5行 | skills/、MCP、rules/ へのポインタ |

## サブディレクトリへの配置

- `cwd` からルートまで階層を**上方向に探索**し、全 `CLAUDE.md` を読み込む
- サブディレクトリの `CLAUDE.md` はそのディレクトリのファイル操作時に**オンデマンド読み込み**
- モノレポ対応: 言語・ドメインごとに `src/CLAUDE.md`, `api/CLAUDE.md` 等で分割可能

```
monorepo/
├── CLAUDE.md          # 全体共通
├── src/
│   └── CLAUDE.md      # フロントエンド固有（src/ 操作時のみ読み込み）
└── api/
    └── CLAUDE.md      # バックエンド固有（api/ 操作時のみ読み込み）
```

## @ import 構文の詳細

```markdown
## プロジェクト概要
詳細は @README.md を参照。npm コマンドは @package.json を確認。

## Git ワークフロー
@docs/git-instructions.md
```

| 仕様 | 内容 |
|------|------|
| パス形式 | 相対パス・絶対パス両対応 |
| ホームディレクトリ | `@~/.claude/...` で参照可能 |
| 再帰インポート | 最大深さ **5 ホップ** |
| コードスパン内 | `` `@file` `` はインポートされない（評価スキップ） |
| 初回承認 | 外部ファイルのインポートは初回のみ確認ダイアログ |

**効率的な使い方**: 大きなファイルを丸ごと `@import` するより、要約を CLAUDE.md に書く方がトークン効率が良い。

## Auto Memory

Claude Code がセッションを通じて自動的に学習内容を記録する機能。

| 項目 | 内容 |
|------|------|
| ロケーション | `~/.claude/projects/<project>/memory/MEMORY.md` |
| 記録内容 | デバッグ知見、アーキテクチャノート、プロジェクトパターン |
| 管理 | `/memory` コマンドで閲覧・編集 |
| 無効化 | `CLAUDE_CODE_DISABLE_AUTO_MEMORY` 環境変数 |

---

## # キーによるクイックメモリ追加

セッション中に `#` キーを押すと即座に CLAUDE.md を編集できる。作業中に気づいたルールをその場で記録可能。追加した内容はセッション終了後も永続化される。

---

## /memory コマンド

```bash
/memory    # システムエディタで全メモリファイルを一括編集
```

複数のメモリファイル（CLAUDE.md、CLAUDE.local.md、Auto Memory）を統一インターフェースで管理。

## ポインタ型設計

CLAUDE.md は「説明文書」でなく「ポインタ」として設計する。

**書くべきもの:**
- ルーティング指示: `npm test` で走る、ADR は `/docs/adr/` にある
- 禁止事項一覧（各項目が ADR またはリンタールール参照を持つ）
- ビルド・テスト・デプロイの最小限コマンド

**書くべきでないもの:**
- システム現状説明（コードとテストが真実のソース）
- 技術スタック解説（エージェントは package.json や go.mod を読める）
- 冗長なコーディングスタイルガイド（リンター・フォーマッターに委ねる）

**利点**: ポインタが指すパスが存在しなければ 404 相当のエラーが出る。記述ドキュメントの腐敗は沈黙で進行するが、壊れたポインタは騒がしく失敗する。

## ユーザーレベルルール（補足）

`~/.claude/rules/` に個人ルールを配置可能。プロジェクトルールより先に読み込まれる。

```
~/.claude/
├── CLAUDE.md        # 全プロジェクト共通設定
├── rules/           # 個人ルール（パスマッチでオンデマンド）
│   ├── git.md
│   └── code-style.md
└── skills/          # 個人スキル
```
